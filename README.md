# EmoFlow

**EmoFlow** — веб-приложение для развития навыков продуктивной психологической переработки негативных эмоциональных состояний.

Приложение разработано в рамках комплексной дипломной работы и предназначено для самостоятельного прохождения пользователем 21-дневной программы психологических упражнений. EmoFlow помогает фиксировать эмоциональные состояния, анализировать ситуации, отслеживать прогресс, выполнять упражнения на осознанность, переоценку, принятие и саморегуляцию.

Проект не является медицинским или клиническим инструментом и не заменяет консультацию специалиста. Его назначение — предоставить пользователю удобную цифровую среду для регулярной работы с эмоциональными состояниями и формирования навыков саморефлексии.

## Возможности

- 👤 **Регистрация и авторизация**  
  Пользователь может создать аккаунт, войти в систему и работать со своими индивидуальными данными.

- 📅 **21-дневная программа упражнений**  
  Программа состоит из последовательных дней, каждый из которых содержит психологические упражнения.

- 🔒 **Последовательное открытие дней**  
  Первый день доступен сразу. Следующие дни открываются после завершения предыдущих.

- ✍️ **Сохранение пользовательских ответов**  
  Ответы сохраняются в базе данных и подставляются обратно при повторном открытии упражнения.

- 🧠 **Разные типы упражнений**  
  В приложении используются текстовые ответы, шкалы, выбор одного варианта, множественный выбор, поля для собственного ответа, информационные блоки, дыхательные упражнения и аудиомедитация.

- 🌬️ **Дыхательные практики**  
  Для отдельных упражнений добавлена визуальная анимация дыхания с движущейся точкой по траектории.

- 🎧 **Аудиомедитация**  
  В упражнение с аудиомедитацией добавлен спокойный звуковой фон `calm-rain`.

- 🎨 **Элемент рисования**  
  Для упражнения с картой эмоций предусмотрен элемент рисования, который позволяет пользователю визуально выразить свое состояние.

- ⭐ **Баллы и прогресс**  
  За завершение дней пользователь получает баллы. Прогресс отображается на странице программы и на отдельной странице прогресса.

- 📊 **Диаграммы прогресса**  
  В приложении отображается визуальная диаграмма прохождения программы, а также круговая диаграмма на странице прогресса.

- 🧾 **Итоговая диагностика**  
  После завершения 21-го дня открывается итоговая диагностика, включающая полный набор диагностических вопросов.

- ✅ **Итоговое заключение**  
  После прохождения диагностики пользователь получает общий вывод с кратким итогом, подробной расшифровкой и важным уточнением о неклиническом характере результата.

- 📱 **Адаптивный интерфейс**  
  Интерфейс адаптирован под разные устройства: компьютер, ноутбук, планшет и смартфон.

## Основная логика приложения

EmoFlow построен вокруг постепенного прохождения программы. Пользователь регистрируется, входит в аккаунт и попадает на страницу программы. На этой странице отображается список дней, общий прогресс и количество набранных баллов.

Каждый день содержит несколько упражнений. Упражнения открываются последовательно, а после выполнения всех заданий день считается завершенным. После завершения дня пользователь возвращается к программе, где видит обновленный прогресс.

Промежуточная диагностика после 14-го дня удалена. После завершения 21-го дня пользователь переходит к новой итоговой диагностике. Результат диагностики выводится на отдельной странице, чтобы форма диагностики и результат не смешивались друг с другом.

## Архитектура

Приложение реализовано на основе Django и имеет классическую структуру серверного веб-приложения.

```text
Пользователь
    ↓
Браузер
    ↓
Django-приложение
    ├── маршруты urls.py
    ├── представления views.py
    ├── модели models.py
    ├── формы forms.py
    ├── шаблоны templates/
    ├── статические файлы static/
    └── база данных SQLite
```

Основные части приложения:

- **models.py** — описание структуры данных;
- **views.py** — логика обработки страниц и пользовательских действий;
- **urls.py** — маршруты приложения;
- **forms.py** — формы регистрации и обработки данных;
- **templates/** — HTML-шаблоны страниц;
- **static/** — стили, изображения, аудио и JavaScript;
- **management/commands/** — команды для заполнения программы и итоговой диагностики.

## Быстрый старт

### 1. Клонировать или открыть проект

Перейти в папку проекта:

```bash
cd D:\NCFU\emotion_webapp
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv
```

### 3. Активировать виртуальное окружение

Для Windows:

```bash
.venv\Scripts\activate
```

### 4. Установить зависимости

Если в проекте есть файл `requirements.txt`:

```bash
pip install -r requirements.txt
```

Если файла нет, минимально необходимая установка:

```bash
pip install django
```

### 5. Применить миграции

```bash
python manage.py makemigrations
python manage.py migrate
```

### 6. Заполнить программу упражнениями

```bash
python manage.py seed_program
```

### 7. Заполнить итоговую диагностику

```bash
python manage.py seed_final_diagnostic
```

После выполнения команды в терминале должно появиться сообщение о создании полной итоговой диагностики.

### 8. Создать администратора

```bash
python manage.py createsuperuser
```

### 9. Запустить сервер

```bash
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000/
```

## Работа с базой данных

На этапе разработки используется SQLite. База данных хранится в файле:

```text
db.sqlite3
```

Основные данные, которые сохраняются в базе:

- пользователи;
- дни программы;
- упражнения;
- вопросы упражнений;
- ответы пользователей;
- прогресс прохождения;
- вопросы итоговой диагностики;
- ответы итоговой диагностики;
- итоговые результаты пользователя.

Если после изменения моделей появляются ошибки, связанные с отсутствием столбцов, необходимо выполнить:

```bash
python manage.py makemigrations
python manage.py migrate
```

Если данные не являются важными и нужно полностью пересоздать базу, можно удалить файл:

```text
db.sqlite3
```

После этого нужно заново выполнить миграции и команды заполнения:

```bash
python manage.py migrate
python manage.py seed_program
python manage.py seed_final_diagnostic
python manage.py createsuperuser
```

## Заполнение программы упражнениями

Основная программа упражнений заполняется командой:

```bash
python manage.py seed_program
```

Команда создает структуру 21-дневной программы:

- блоки программы;
- дни;
- упражнения;
- вопросы;
- варианты ответов;
- специальные задания.

Если в упражнения были внесены изменения, команду можно запустить повторно. При этом важно учитывать, что повторное заполнение может изменить или удалить старые данные программы в зависимости от логики команды.

## Итоговая диагностика

Итоговая диагностика открывается после завершения 21-го дня программы.

Промежуточная диагностика после 14-го дня удалена. Старая итоговая диагностика после 21-го дня также заменена новой итоговой диагностикой.

Новая итоговая диагностика включает полный набор вопросов:

```text
34 вопроса — методика диагностики субъективного благополучия
10 вопросов — опросник эмоциональной регуляции ERQ
12 вопросов — шкала ценности эмоционального контроля ECV
28 вопросов — ССПМ-2020
75 вопросов — 5PFQ
```

Общее количество вопросов:

```text
159 вопросов
```

Для заполнения итоговой диагностики используется команда:

```bash
python manage.py seed_final_diagnostic
```

После прохождения диагностики результат отображается на отдельной странице:

```text
/final-diagnostic/result/
```

Результат включает:

- краткий итог;
- подробную расшифровку;
- важное уточнение о том, что заключение не является клинической оценкой.

Итоговое заключение не делится на видимые для пользователя критерии. Расчет выполняется внутри приложения, но на странице результата пользователь видит цельный и читабельный вывод.

## Адаптивность интерфейса

Приложение адаптировано под разные устройства. На больших экранах некоторые страницы используют двухколоночную структуру: слева размещается описание упражнения, справа — поля ответов.

На мобильных устройствах блоки перестраиваются вертикально. Это позволяет проходить программу со смартфона без горизонтальной прокрутки и потери читаемости.

Адаптированы следующие элементы:

- шапка сайта;
- главная страница;
- регистрация и вход;
- список дней;
- карточки упражнений;
- формы ответов;
- страница прогресса;
- итоговая диагностика;
- страница результата;
- дыхательная анимация;
- блок аудиомедитации.

## Безопасность

В приложении используются базовые механизмы безопасности Django:

- авторизация пользователей;
- разграничение доступа к личным страницам;
- CSRF-защита форм;
- хранение паролей средствами Django;
- доступ к упражнениям, прогрессу и диагностике только после входа в аккаунт.

Пользовательские ответы хранятся в базе данных и привязаны к конкретному аккаунту. Другие пользователи не имеют доступа к этим данным через интерфейс приложения.

Перед публикацией проекта на сервере рекомендуется дополнительно:

- отключить режим отладки `DEBUG`;
- настроить `ALLOWED_HOSTS`;
- использовать HTTPS;
- защитить административную панель;
- регулярно выполнять резервное копирование базы данных;
- ограничить доступ к файлу базы данных;
- не публиковать секретные ключи и личные данные пользователей.
